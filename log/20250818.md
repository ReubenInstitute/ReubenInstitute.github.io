The architecture from the previous session became code. The focus was implementing `Psalm` and `BibleArticle` inside the unified Scriptures platform, starting with the indexing war that had to be won first.

### The indexing fix

Off‑by‑one errors between 1‑based biblical numbering and 0‑based array indexing had been causing silent chaos. The solution was pure 0‑based indexing throughout, stripping away the confusing `_by_number` and `_by_index` dictionaries in favour of clean array access:

```python
# OLD (confusing mix):
bible[1]         # Book number 1? Or index 1?
bible.books[0]   # Index 0

# NEW (consistent):
bible[0]         # Always index 0 (Genesis)
bible[11]        # Always index 11 (Isaiah)
```

### BibleArticle class

A new class for curated, formatted Bible passages with user‑customisable Markdown:
* Auto‑creates Markdown files from Bible text if they don’t exist.
* Uses verse prefixes: `[versenr][space]` (e.g. “1 In the beginning…”).
* Supports user formatting with paragraph breaks via blank lines.
* Parses Markdown into the segments‑and‑verses hierarchy defined in the parsing specification.

### Psalm class enhanced

The `Psalm` class was cleaned up, indexing was made consistent, and it now works seamlessly with the Bible data structure following the same pattern as `BibleArticle` — user‑customisable Markdown formatting over a canonical text layer.

### Robust data loading

CSV handling was hardened: proper `NaN` and empty‑value management with sensible defaults, comprehensive validation and error reporting, and efficient download and caching of source texts.
