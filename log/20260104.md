# 4 Jan 2026 – DeepSeek 3 – Clipboard feature for Psalm descriptions

_Studio_

A one‑click clipboard feature was added to the Psalm page so that formatted YouTube descriptions, social‑media blurbs, and other templates can be copied instantly without ever leaving the studio.

### Back‑end

A single `descriptions.md` file now holds all templates, keyed by `##` headers.  The Flask route parses this file on load, renders each template through Jinja2 with the current Psalm’s context, and builds a dictionary mapping description types to their rendered text.  The logic is minimal — about 15 lines added to the route.

### Front‑end

Buttons are generated dynamically, one per description type.  The rendered descriptions are stored in a JavaScript object (properly escaped on the server side), and a `clipboard(name)` function writes the selected text to the system clipboard with `navigator.clipboard.writeText()`.  No external libraries, no build step.

### Design

The system is scalable — add a new description type by simply adding a header and template to the Markdown file.  The templates remain external and human‑editable; the code stays unchanged.  One click, instant copy, zero friction.
