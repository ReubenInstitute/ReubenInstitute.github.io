# 28 Mar 2026 – DeepSeek 3 – Flask app refactored with Booklets.py data model

_Code, HistoryBooklets_

The booklet series had been living as raw Markdown files served through a Flask app with inline templates and scattered file‑handling logic.  We replaced all of that with a clean data layer.

Two core classes were built in `Booklets.py`.  `TraditionBooklet` represents a single booklet—its metadata, language availability, and text content, with properties `english_text`, `hebrew_text` and later `english_html`, `hebrew_html` that encapsulate all file I/O and Markdown‑to‑HTML conversion.  `TraditionCollection` loads every booklet from a CSV and provides lookup by basename, with `__getitem__` for direct access.  All file operations use `pathlib.Path` with constants stored as `Path` objects.

The Flask app was then gutted of its inline logic.  Templates were extracted to separate files (`base.html`, `index.html`, `booklet.html`), routes were rewritten to use `TraditionCollection` and the properties of `TraditionBooklet` exclusively, and PDF generation was updated to consume `english_html`/`hebrew_html` directly.  The app no longer touches a file on disk.

A `core` boolean field was added to `booklets.csv` to distinguish annual observances (holidays, fast days, Shabbat) from life‑practice topics.  The documentation was updated to include this distinction with star‑emoji markers in the collection table, polished language throughout, and a full API reference for the two new classes.

The result was a clean separation of concerns: `Booklets.py` owns all data and rendering; `app.py` handles only requests and presentation.  The system is ready to serve the existing series and trivially extensible to others.
