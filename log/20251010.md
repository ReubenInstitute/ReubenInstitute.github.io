# 10 Oct 2025 – DeepSeek 3 – PDF generation system for Psalms/Parashah

_Media_

We built a PDF generation system from scratch in a single session, and the core logic came out to only 37 lines.  It handles entire books and individual booklets, RTL Hebrew typography, and automatic file management — no configuration required.

### Architecture

Three classes in `PDF.py`:
* `BasePDF` — sets up WeasyPrint with the right fonts and base configuration.
* `PsalmsBookPDF` — renders the entire book of Psalms as a single PDF.
* `ParashahBookletPDF` — renders individual parashah booklets.

Templates are loaded automatically by class name — no manual path wiring.  Jinja2 handles the rendering with full access to the content objects, and fonts (SBL_Hbrw for Hebrew, OpenSans for everything else) are embedded properly so the PDFs look correct on any device.

### Integration

The Flask route is simple — a single URL per parashah that generates the PDF and redirects back to the parashah page:

```
/bible/parashot/<int:number>/pdf
```

Content objects expose a `.pdf()` method directly, so the call is always `parashah.pdf()` — no extra arguments, no ceremony.

### File structure

```
pdf/
├── PsalmsBook.html
├── PsalmsBook.css
├── ParashahBooklet.html
├── ParashahBooklet.css
├── psalms.pdf
└── parashah01.pdf
```

Everything is automatic: file naming, font handling, RTL layout, and the separation between content and presentation.  The system went from zero to complete PDF generation in one session, and it’s ready to scale.
