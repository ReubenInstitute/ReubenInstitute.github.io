# 9 May 2026 (b) – DeepSeek 3 – Audio pipeline restored, video sync fixed, Scriptures library extracted

_Code, Studio, Media, Libraries_

We resurrected the broken Studio by systematically replacing every legacy `bible.audio` call with the new `AudioBible` API, then forced the video pipeline to consume the pre‑built `PsalmAudio` streams for perfect sync.  After that, the session branched into a publishing‑plan detour, a fast extraction of the Suno Browser and Psalm Media Studio, and finally the separation of the entire text‑corpus layer into a standalone `Scriptures` package.

### Audio restoration

The main Studio (`studio.py`) had been gutted: all audio‑duration calculations were commented out, and the old `bible.audio` object no longer existed.  A tiny `test.py` script proved `AudioBible` worked flawlessly with verse objects.  We then:

* Restored the index route with a single structured `stats` dictionary (no flat‑variable explosion), computing original/cloned/English counts and durations for Psalms and Parashot.
* Rebuilt the `/bible/psalms` list page with per‑psalm cloned durations, per‑book totals, and grand totals, all drawn directly from `AudioBible`.
* Fixed the psalm detail page by passing an `audio` object to the template and calling `audio.has_original_audio(verse)` etc. directly, exactly as the old template expected but with the new signature.
* Discovered the mini waveform players weren’t appearing because the correct CSS class was `simple-player` (not `visual-player`).  The real `audio.js` already auto‑initialised all `.simple-player` elements; no new init code was needed.

The edit pages and the audio‑marking editor were also updated to use `AudioBible` and the new verse‑based `get_timing`/`set_timing` methods.

### Video sync

`PsalmVideo` was still assembling its own audio stream separately from `PsalmAudio`, causing consistent drift between image fades and narration.  We killed the duplicated audio assembly and made `PsalmVideo` consume the pre‑built `PsalmAudio` streams:

* `PsalmVideo.duration` now simply returns `self.psalm.audio.duration` (via a local `PsalmAudio` instance).
* `PsalmVideo.audio_stream` returns `psalm_audio.stream` or `enhanced_stream`, depending on whether background music exists.
* The title video segment, however, was only as long as the bare title audio, missing the surrounding fades that `PsalmAudio` includes.  Adding `FADE_DURATION + title_dur + FADE_DURATION` to the title video duration fixed the sync completely.

Regular, raw, horizontal, and vertical Psalm videos now export with frame‑perfect audio‑video alignment.

### Suno Browser and Psalm Media Studio

Two small Flask apps were carved out of the existing media‑browser code, sharing only the `Suno.py` library:

* **Suno Browser** (`suno-browser.py`) — the general listening tool for all three libraries (`robertai`, `psalms`, `piyutim`).  Library paths were moved to a `libraries.json` config.  Pinning remains a simple “favourite” bookmark, not a psalm assignment.
* **Psalm Media Studio** (`psalm-media-studio.py`) — a dedicated browser for the `psalms` Suno library, using its own `templates-psalms/` folder.  No pinning, no project logic; just browsing songs with waveform players, Suno links, and creation dates.  Assignment will be added next.

Both apps run on separate ports and share the same underlying audio/image files.

### Scriptures library extraction

When the Psalm Media Studio needed access to the Bible data classes, we pulled all 13 core text‑corpus modules (`Bible.py`, `BiblePsalms.py`, `BibleParashot.py`, `Mishnah.py`, `TalmudBavli.py`, `TalmudYerushalmi.py`, `Zohar.py`, `ZoharChadash.py`, `ZoharTikkunim.py`, `Scriptures.py`, `Text.py`, `Hebrew.py`, `HebrewNumbers.py`) plus the `db/` and `text/` data folders into a standalone `/sdcard/Scriptures` package.  Every hard‑coded relative path (`"db/bible.json"`, `"text/psalms.md"`, etc.) was replaced with `ROOT / "db/bible.json"` etc., using `ROOT = Path(__file__).parent`.

The package is made importable via `PYTHONPATH=/sdcard/Scriptures:$PYTHONPATH`.  The main Studio now imports everything from `Scriptures` and works correctly, with one remaining path fix in `TextStore` (still looking for `"text"` in the CWD instead of `ROOT / "text"`).  The Hebrew sub‑library and the audio layer are slated for similar extraction in future sessions.
