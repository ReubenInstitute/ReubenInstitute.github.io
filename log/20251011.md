# 11 Oct 2025 – DeepSeek 3 – Media slide system with $ markers

_Media_

The platform needed a way to split long content into individual visual slides without forcing editors to manually define every break.  The solution was elegantly simple: a single `$` character in a verse’s text marks where a new slide should begin.

### The slide class hierarchy

Three slide classes were built in `Slide.py` — `ParashahSlide`, `ArticleSlide`, and `PsalmSlide` — each with proper hierarchical numbering that reflects the content’s natural structure.  The API is consistent across all types: a slide knows its number, its parent paragraph, and how to save itself as an image.

### Image generation and processing

`Image.py` was updated to accept slide objects directly.  It generates images for every content type and organises them into a clean file structure: `build/images/parashot/`, `build/images/psalms/`, `build/images/articles/`.

The processing logic itself is straightforward.  During content loading, the system scans verse text for the `$` character.  When it finds one, the verse is split: content before the marker stays in the current slide, content after starts a new one.  The markers are removed from the final text, so they never appear in rendered output.  Crucially, the existing `.layout` property was preserved alongside the new `.slides` property — no breaking changes, full backward compatibility.

### Files updated

* `BibleParashot.py` — added `ParashahSlide`, integrated media‑slide processing, and fixed a paragraph duplication bug where episodes were showing their content twice.
* `BibleArticles.py` — added `ArticleSlide`, implemented sequential article‑level numbering, and wired in slide processing.  A CSV loading bug with `iterrows()` returning tuples was caught and fixed during the integration.
* `BiblePsalms.py` — structure prepped and ready for implementation; the pattern is proven, just needs the processing loop.

### Critical bugs fixed

Three problems were resolved in this session alone:
* Paragraph duplication — episodes displaying their content twice due to a double‑creation loop.
* CSV loading — `iterrows()` tuple unpacking that silently corrupted row data.
* Media slide integration was added without destabilising any existing functionality.

The naming is clean, Pythonic, and follows the project’s strict no‑Java‑prefixes rule.  The hierarchical slide numbering system works consistently across content types, and the whole system is backward‑compatible.  The foundation is ready for the next phase: overlays and specialised renderers.
