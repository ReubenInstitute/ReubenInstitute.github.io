# 2 Jan 2026 – DeepSeek 3 – Web‑based video editor for Psalm videos

_Studio_

A minimal, browser‑based video editor was built from scratch so that every Psalm could get a custom‑stitched background sequence with professional transitions.

### The timeline engine

A new `Editor.py` module introduced a project system with 150 projects — one per Psalm — each managing a list of source video files as `ClipItem` objects.  Every clip carries a timestamp, offset, and duration.  The engine enforces two hard constraints: zero gaps (no empty spaces between clips), and at most two overlapping clips at any moment.  When it’s time to export, FFmpeg assembles the timeline with fade‑through‑white between adjacent clips, crossfades where they overlap, and automatic gap‑filling if needed.

### Web interface

A Flask app (`editor-app.py`) provides a project browser listing all 150 Psalms, a timeline view that renders clips as coloured blocks, an asset management pane with thumbnails, and an export button that kicks off the final video.  The back‑end serves video segments through an endpoint like `/project/<number>/segment/<start>-<end>.mp4`, which currently returns black test videos but is architecturally ready for real timeline rendering via FFmpeg filter graphs.

### Streaming player

The front‑end player splits playback into 10‑second chunks.  It preloads the next segment five seconds before the current one ends, streams them seamlessly, and dispatches custom events (`timeupdate`, `segmentchange`, `ended`) so other UI components can react.  Controls are keyboard‑driven — space for play/pause, timeline click to seek — with a real‑time time display.

### Accidental verse‑timing system

While working on the editor, we inadvertently built a complete verse‑timestamp infrastructure into `BibleAudio`.  A `timestamps.csv` file now maps every `(book, chapter, verse)` to a moment in time, with `timestamp()` and `set_timestamp()` methods and an automatic CSV loader.  It’s not wired into the current editor — it’s future‑proofing for audio‑synchronised video generation — but the foundation is ready.

### What’s working and what’s not

Loading any project from 1 to 150 works.  Segmented black video streams and play/pause/seek function correctly.  Real video rendering, timeline drag‑and‑drop, and clip adjustment methods are not yet implemented.  The architecture — event‑driven, progressively enhanced, with clean class separation — is designed to make those additions straightforward.
