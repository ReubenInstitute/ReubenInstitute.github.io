# 20 Oct 2025 – DeepSeek 3 – PIL text rendering height and line spacing fixed

_Font_

Two bugs were conspiring to break Hebrew text rendering across the entire platform, and they both traced back to the same root cause: over‑engineered positioning logic that tried to outsmart PIL instead of working with it.

### The height mismatch

`Image.bbox()` and `Image.text()` were returning completely different results for the same text.  `bbox()` would report 68 px height — correct for the Hebrew characters with their diacritics.  But `text()` would draw that same string and it would appear 500 px tall, wildly misaligned with everything around it.  The cause was a complex temp‑image logic inside `text()` that used wrong positioning offsets.

The fix was to make `text()` use the same extreme‑character logic as `bbox()`.  We create a temp image with the exact dimensions that `bbox()` calculated, draw the text at `(0,0)` inside it — no offsets, no translations — and auto‑clip the extreme characters by positioning them outside the image bounds.  The result is then pasted at the user’s target coordinates.  One consistent method for measurement and drawing, zero positioning drift.

### Inconsistent line spacing

Throughout the slide generation code, a magic `1.3` multiplier was being applied to line spacing without any rationale.  It had been there so long nobody questioned it.  The fix was simple: remove it.  Line spacing is now derived from the font’s actual metrics, consistent across every rendering class.

### Mixed bounding‑box methods

Some code paths used `Image.bbox()`, others used `canvas.textbbox()`, and the two methods return slightly different measurements for the same text.  We standardised on `Image.bbox()` everywhere — one method, one source of truth for Hebrew text measurement.

### Key lessons

PIL has no concept of a text baseline — you draw from the top coordinate only.  Extreme characters (the highest cantillation mark, the lowest vowel point) are necessary to calculate proper Hebrew diacritic height.  Temp images must match the bounding‑box dimensions exactly.  Auto‑clipping extreme characters is cleaner than erasing them manually.  And consistency between measurement and drawing is non‑negotiable.

### Files modified

`Image.py` was updated with the fixed `Image.text()` method and corrected slide save functions.  `test.py` was extended with a comprehensive test that verifies height consistency between `bbox()` and the actual rendered output.

The core insight was that the original code had built a Rube Goldberg machine of positioning calculations when the actual solution was far simpler: make `text()` draw inside the exact same bounds that `bbox()` calculates.  Now it does.
