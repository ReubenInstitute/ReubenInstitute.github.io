# 2 Oct 2025 – DeepSeek 3 – Unified audio system with original/cloned/English

_Media_

We designed and implemented a comprehensive audio handling system that serves Psalms, Articles, and Parashot through a single, unified pipeline.  Each content type now has access to original Hebrew, AI‑cloned Hebrew, and English text‑to‑speech, with exports in both MP3 and telephony‑ready SLN formats.

### Architecture

The system is split into two clean layers.  `Audio` handles file management — transparent MP3‑to‑WAV conversion, CSV‑based verse‑timing lookups, and export methods for every format.  `AudioAI` handles generation — voice cloning and text‑to‑speech via ElevenLabs, with automatic duration tracking persisted to CSV.  The two classes communicate through a shared cache but never tangle responsibilities.

Verse‑level integration was achieved by adding an `audio` property to the existing `Bible.Verse` class, coupled with an `english_text` field.  Application‑level classes avoid circular imports by storing a reference to the real `Bible.Verse` and delegating:

```python
class AppVerse:
    def __init__(self, app_container, text, number):
        self.app_container = app_container
        self.text = text
        self.number = number
        
        book = self.app_container.bible[book_num]
        chapter = book[chapter_num]
        self.bible_verse = chapter[verse_num]  # Key to avoiding circular imports
    
    @property 
    def audio(self):
        return self.bible_verse.audio
```

### Performance

The audio‑timing CSV spans over 71 000 rows.  Loading it repeatedly would have been a bottleneck, so the data is cached once at the class level.  The existing API was preserved entirely — the caching is transparent, and verse lookups are now instant.

### What was completed

* `Audio` class with file management, export methods (`export_original_mp3()`, `export_cloned_sln()`, `export_english_sln()`), and status properties for file existence and duration.
* `AudioAI` class with ElevenLabs integration (`clone()`, `generate_english()`) and automatic CSV persistence.
* Flask audio‑serving routes at `/audio/original/`, `/audio/cloned/`, and `/audio/english/`.
* A minimal statistics display on the main page.

The key architectural rule going forward: when application‑level classes need core Bible functionality, use explicit composition with `bible_verse` delegation — never inheritance — to keep the import graph clean.
