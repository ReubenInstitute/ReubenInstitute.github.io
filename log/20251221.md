# 21 Dec 2025 – DeepSeek 3 – Audio editing enhancements and homepage statistics

_Studio_

The Bible multimedia production system received targeted upgrades to the audio editor and the statistics display, making daily production work faster and project tracking more informative.

### Audio editing improvements

Keyboard shortcuts were added to the waveform editor, eliminating the need to click tiny handles for precise adjustments.  The arrow keys move markers in fine (0.1 second) and coarse (1 second) increments.  The `S` key toggles focus between the start and end markers, and the spacebar controls play/pause.  The waveform container auto‑focuses on load so keyboard control is immediate — no click required.

A version‑specific WaveSurfer.js bug was discovered and fixed: our particular build requires `region.setOptions()` instead of the documented `region.update()` for updating marker positions.  The available visual customisation options were documented — `waveColor`, `progressColor`, `cursorColor`, `cursorWidth`, and bar‑style options (`barWidth`, `barGap`, `barRadius`, `barHeight`, `barAlign`) plus layout controls (`height`, `width`, `minPxPerSec`, `fillParent`).

### Homepage statistics overhaul

The old display showed only aggregate Psalm audio counts, which was too coarse for tracking production across multiple series.  It was replaced with separate tracking for Parashot (Torah portions) and Psalms, each showing counts with icons — 🔊 Original, 👥 Cloned, 🇺🇸 English — and cumulative durations formatted intelligently: `MM:SS` for under an hour, `H:MM:SS` for anything longer.  The durations now reflect the total audio time for each category and type, giving an instant picture of how much content is ready and where the gaps are.

The approach throughout was minimal — only the changes that mattered, preserving all existing functionality, and adapting precisely to the version of WaveSurfer.js in use.
