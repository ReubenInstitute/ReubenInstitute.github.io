# 8 Oct 2025 – DeepSeek 3 – CSV naming standardized and build system refactored

_Code_

A routine data cleanup turned into a deeper lesson about simplicity when we tried to refactor the build system and nearly broke everything.

### The discovery

CSV files across the project had inconsistent field naming: `book_number` in one place, `book` in another; `num_verses` versus `verses`.  Some files stored combined verse strings like `"1.17"` that needed to be split into separate chapter and verse columns.  It was the kind of drift that creeps in when a project grows organically and nobody has audited the metadata layer.

### The standardization

We brought every CSV into alignment.  `episodes.csv` was reordered, columns renamed, verse data split.  `parashot.csv` followed the same treatment.  The consistent naming convention — `english_name`, `hebrew_name`, `book`, `verses` — was applied everywhere.  `BibleParashot.py` load functions were updated to parse the new formats, and both `app.py` and `studio.py` were modified to display Hebrew names instead of English.  Test scripts confirmed the data loaded correctly: 54 parashot, 115 episodes.

### The refactoring detour and its collapse

We then attempted to refactor `build-sources.py` into a `Source` class.  It failed.  The instinct to build a complex config system took over, and within hours we had introduced bugs with hardcoded indices and directory path issues.  The fix wasn't to debug the complexity — it was to remove all of it and return to simple, direct file access.  The original script had worked because it was straightforward; our refactoring failed precisely when we added unnecessary abstraction, and it succeeded again when we stripped that abstraction back out.

### The data integrity crisis

While working on the build, we discovered that the Zohar chapter metadata CSV had 30 rows — but the actual data contained 53 sections.  The mismatch had been silently present, waiting to cause problems downstream.  The fix was expanding `zohar-chapters.csv` to include every missing section and resolving a path confusion between `cls.CACHE_DIR` and `CACHE_DIR`.  Once those were aligned, the `Source` class replicated the original script's functionality exactly, with all cleaning and verification logic preserved.

### The lesson

Simple, direct solutions beat over‑engineered abstractions every time.  We learned it twice in one session: once when complexity broke the build, and again when removing it fixed everything.
