# 20 Oct 2025 – DeepSeek 3 – Font metrics system with fontTools and Unicode

_Font_

The deeper we went into Hebrew typography, the more we collided with a fundamental limitation: PIL has no concept of a text baseline, and for complex scripts like Hebrew with diacritics, that means inconsistent line spacing and misaligned text.  We needed to stop relying on PIL’s broken measurements and go straight to the font files themselves.

### Font metrics foundation

We used `fontTools` to extract the true typographic metrics directly from the font binaries.  Every font now exposes its real `ascent`, `descent`, `linegap`, and `units_per_em`, scaled to the font size.  Line spacing is calculated the way a proper typesetting engine would: `ascent + descent + linegap`.  No more guesswork, no more magic multipliers.

### Extreme character detection

To handle the full vertical range of Hebrew, we identified the characters with maximum extent in both directions.  The top extreme is a Hebrew letter combined with the highest cantillation mark; the bottom extreme is a letter with the lowest diacritic or vowel.  These are stored as `extreme_chars` and used for stable height calculations across every rendering class.

### Comprehensive Unicode system

A complete Hebrew Unicode database was built, covering 37 letters (including final forms and alternatives), 17 diacritics and vowel points, 35 cantillation marks, and extensive punctuation sets.  The API is clean and accessible: `unicode.hebrew_letters`, `unicode.diacritics`, and so on.  Any rendering code can now query the character database without hardcoding a single code point.

### Fixed Font class

Each font variant pre‑calculates its metrics at initialisation, so there’s no more runtime measurement overhead.  The metrics are consistent and accurate regardless of which PIL function is calling them.

### What’s ready for the next session

With the font metrics and character database complete, we can now update `Image.bbox()` and `Image.text()` to use real typographic data, fix all the rendering classes, and implement consistent line spacing everywhere.  The unstable text positioning that’s been haunting the platform is finally within reach of a permanent fix.
