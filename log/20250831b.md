# 31 Aug 2025 – DeepSeek 3 – Akiva & Rachel video: storyboard, character bible, Kling pipeline

_Media_

We took the Talmudic text from Berakhot 62b – the story of Rabbi Akiva’s transformation from an illiterate shepherd to a great sage, driven by his wife Rachel’s faith and sacrifice – and turned it into a fully structured, pre‑production‑ready video plan.

### Production framework

The workflow is standardised: Midjourney generates two key images per scene (start frame and end frame), then Kling animates the transition between them based on a written video prompt.  This gives us frame‑accurate control over every visual beat.

### Segment breakdown

We storyboarded the entire narrative into verses, with a header and blockquote for each segment of the Talmudic text.  The result is a clear, scene‑by‑scene structure that can be handed directly to production.

### Character bible

All characters were identified – Main (Akiva, Rachel) and Supporting (Kalba Savua and others) – and a naming convention was locked in (CamelCase, e.g. `YoungAkiva`, `AgedRachel`).  For each main character we wrote detailed, objective physical descriptions focused on immutable features: face shape, eye type, nose, and build.  Age‑related modifications (Young, Mature, Aged) are defined only for attributes that actually change: age, hair/beard colour, clothing, posture, and expression.

### Asset creation strategy

A two‑phase process generates consistent character images:
* **Phase 1 – Base Reference Sheets**: each character is generated on a plain background in a neutral pose, locking in their core appearance.
* **Phase 2 – Pose‑Specific Images**: the base images are fed back into Midjourney as inputs to generate the same character in the specific poses required for each scene.

The session moved us from a raw Talmudic text to a complete production pack: a scene list, a character bible with detailed visual descriptions, and a repeatable technical workflow for generating consistent assets.  The foundation for producing the video is ready.
