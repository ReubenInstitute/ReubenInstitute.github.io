# 19 Mar 2026 – DeepSeek 3 – System‑wide refactor: Asset base class and media architecture rebuild

_Code, Media_

This was the session where the media architecture finally got a single, clean size model.  We introduced an `Asset` base class that holds `size`, `width`, `height`, and a `landscape` boolean as the single source of truth for dimensions, eliminating the scattered resolution constants and duplicated size logic that had been accumulating across the codebase.  `Asset` and resolution constants moved into `Media.py`, and the vestigial `Content.py` was deleted.

The visual hierarchy was redesigned around four clean abstractions: `Image` for single images, `Plate` for text rendering, `Slide` as a container for plates, and `Video` for composition.  The `Word` class moved to `Text.py` where it belonged.  Every legacy slide and overlay class was replaced with modular, extensible plates and slides — `PsalmVersePlate`, `EpisodeParagraphSlide`, and their siblings — that could be composed without fighting an inheritance chain.

The Bible data classes — Psalms, Parashot, and their containers — were refactored into pure data holders.  All size attributes and slide‑generation logic were stripped out and moved into the presentation layer where they belonged.  The audio pipeline was optimised by caching ElevenLabs title durations in a `titles.csv` file, eliminating the runtime `ffprobe` calls that had been making psalm listing pages slow.  After this session, nearly every core file in the platform had been touched, and the foundation for word‑level highlighting and multi‑format exports was solid.
