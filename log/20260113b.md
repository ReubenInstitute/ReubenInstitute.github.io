# 13 Jan 2026 – DeepSeek 3 – Delta system and timeline refactoring

_Studio_

The timeline system in the declarative video composition engine was redesigned to match how humans actually think about sequences — sequentially, not with absolute timestamps.

### The timestamp problem

Requiring a `timestamp` field for every item in the timeline was cumbersome and unnatural.  Compositors don’t think in absolute seconds from zero; they think in terms of “after that clip” or “overlap by two seconds.”

### The delta system

We removed mandatory timestamps entirely.  Items now play sequentially by default, one after the other, with no extra configuration.  A new `delta` property was conceptualised for future use: positive values create gaps, negative values create overlaps.  This keeps the common case simple while leaving the full power available when needed.

### Architecture refactor

The code was restructured into a clean pipeline:
* **Input classes** (`AudioInput`, `VideoInput`) handle source files and stream splitting.
* **Clip classes** (`AudioClip`, `VideoClip`) process individual items.
* **Composition** brings them together for rendering.

The JSON schema was updated to reflect the new design: `timestamp` is no longer required, `delta` was added as an optional property, and `duration` was made truly optional with a minimum of 0 instead of a forced 0.001.

### Testing and validation

A robust testing system was built with JSON schema validation before every test run, comprehensive cases covering audio, video, and mixed media, and immediate failure on validation errors.  The architecture is modular, maintainable, and ready for the features still to come: delta‑based gaps and overlaps, image and text rendering, and more complex transitions.
