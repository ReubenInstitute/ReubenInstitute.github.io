# 2 Oct 2025 – Gemini 2.5 Pro – Parashah annual cycle plan with AI voice

_Parashah, Media_

The annual Torah reading cycle was set to begin on 18 October 2025, and we needed a concrete plan to get Parashah content ready in time.  The strategic core was simple: take the AI‑cloned voice that had already proven itself with Psalms, and apply it for the first time to the narrative Parashah text.  That single decision would unlock a multi‑platform launch.

For the videos, we chose a pragmatic, speed‑focused approach — reuse the familiar “fog background” style and let the high‑quality audio carry the experience, rather than delaying the launch chasing new visuals.  The scope quickly expanded beyond videos: we planned audio‑only exports for the VoiceLine IVR system as `.sln` files, for streaming platforms like Spotify as `.mp3` files, and a print‑ready PDF booklet by adapting the existing WeasyPrint pipeline.

The internal Flask application was formally named the **Production Studio**.  All development work would be a refactoring and migration of existing, working code from the old Psalms app into the new, unified architecture.  The priority was to restore full Psalms functionality first — proving the new engine worked — and then implement Parashah‑specific features on the validated foundation.

### Final action plan

* Refactor the core `AudioAI`, `Audio`, and `Video` classes from the old Psalms app into the new clean architecture.
* Implement the “Poem” text overlay mode and adapt video output for both vertical and horizontal formats, restoring full Psalms video generation.
* Implement the “Narration” text overlay mode for Parashah videos.
* Add audio export functions for `.sln` (IVR) and `.mp3` (Spotify).
* Adapt the existing WeasyPrint pipeline to generate print‑ready PDFs for Parashah booklets.
* Stretch goal: if time allows, extend the print pipeline to also produce a consistent Psalms PDF.
