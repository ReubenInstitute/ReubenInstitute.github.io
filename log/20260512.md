# 12 May 2026 – 📖 DeepSeek 3 – Full pipeline repaired: overlays, shadows, alpha, qubuts exterminated

*Code, Font, Media, PsalmSeries, Studio*

The cover‑generation victory of the previous session was only half the battle.  The same bbox‑measurement bugs had infected every downstream piece of the rendering pipeline, and Psalm 44 would not render cleanly until each one was hunted down.  This second session‑in‑session completed the siege, fixing the overlay, the video class, the alpha‑channel corruption, the clipped shadows, and the phantom diacritics that kept reappearing.

### PsalmOverlay

The overlay used `anchor="mm"` and suffered the same vertical misalignment and clipping as the old covers.  The `draw_text` fix from the previous session (the `-full_bbox[1]` offset) was already in place, but the `anchor="mm"` branch was still drawing `text + extreme_chars` visibly at `draw_x=0`.  We set `draw_x=0` and changed the render call to draw only `text`, matching the logic of the non‑anchor branch.  The overlay became unclipped; remaining Y‑constant drift was corrected by hand via a “screwdriver alignment” technique — tabbing between a TikTok reference and a local render until the three text elements matched.

### PsalmVideo

The video class was still using the deprecated `raw` parameter, the old `-raw` filename suffix, and the obsolete square `back.mp4` for fog backgrounds.  It was updated to:

* `cinematic=False` (default) → Score video, written to `psalms+/psalmXXX.mp4` (or `psalms+/horizontal/psalmXXX.mp4` for landscape).
* `cinematic=True` → Cinematic video, written to `psalms+/cinematic/psalmXXX.mp4`.
* Background for Score now picks the correct size‑specific fog asset (`back-720x1280.mp4` or `back-1280x720.mp4`) instead of the square master.

### Alpha‑channel corruption

Verse slides were being saved as opaque RGB images because a `.convert('RGB')` intended for JPEG exports had bled into the PNG branch.  This caused video fades to black instead of transparent.  The conversion was isolated so it only applies when saving JPEGs; PNG slides regained their RGBA mode.

### Shadow clipping

`add_shadow` created a temporary canvas only 8 px larger than the original alpha channel.  The MaxFilter and GaussianBlur spread the shadow beyond that margin, causing it to be clipped at the edges.  The canvas was expanded with a generous pad, the shadow processed, and then cropped back to the correct size, completely eliminating the clip.

### The qubuts zombie — final kill

The `extreme_chars` measurement scaffolding was being drawn visibly in both the `anchor="mm"` and non‑anchor branches of `draw_text`.  In the non‑anchor branch a negative `draw_x` had been intended to hide it, but the whole approach was fragile.  Both branches were simplified to render **only** the actual `text`, while the canvas sizing still uses the combined `text + extreme_chars` for height.  No more phantom diacritics.

### Result

With all fixes applied, Psalm 44 rendered perfectly: baseline‑aligned verse slides, correct Alpha‑transparent fades, clean shadows, correctly delayed Suno music, and the new file landed precisely in `psalms+/psalm044.mp4`.  Psalm 117 had already validated the pipeline; 44 confirmed it.

The whole text‑rendering and video‑compositing chain is now correct from source to final MP4.  The two‑day siege is over; the machine is ready for the daily schedule.
