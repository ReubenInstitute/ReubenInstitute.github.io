# 16 Mar 2026 – Gemini 3 Pro – Media architecture: pure OOP, composition, unified export()

_Code, Media_

We took a hard look at the media processing pipeline and found a rigid, "bossy" inheritance hierarchy that was actively fighting us.  The fix was a decisive shift to pure object orientation: artifacts owning their own data, composition replacing inheritance, and base classes reduced to dumb render engines.

### The core problem

Filenames and paths were scattered across the codebase, with different classes independently guessing how to format book numbers (`02d` vs `03d`)—no single source of truth.  Worse, the `BaseVideo` and `BaseAudio` classes forced a fixed sequence of events (like alternating video/audio streams) that broke the moment custom logic like title cards or background music was introduced.  The base classes weren't helping; they were constraining.

### The architectural shift

We agreed on a "Single Source of Truth" approach where every artifact object owns its own metadata:
* **Artifacts own paths** — instead of a "Manager" class, each object (a `PsalmSlide`, a `PsalmHebrewVideo`) is responsible for its own file path.  No more scattered string‑formatting logic.
* **Composition over inheritance** — the base classes were gutted.  Derived classes like `PsalmHebrewVideo` now take 100% responsibility for assembling their own internal timelines and streams.  The base class doesn't dictate the sequence; the derived class composes what it needs.
* **Base classes as render engines** — reduced to "dumb printing presses" that only handle the final FFmpeg execution with standardised encoding settings (codecs, bitrates, etc.).  They receive streams as black boxes and render them to the location the object provides.

### Implementation steps

We defined the properties that would standardise the interface between derived and base classes:
* **Standardised streams** — `PsalmHebrewVideo` gained `video_stream` and `audio_stream` properties that return finalised FFmpeg nodes.  Assembly and mixing logic moved out of the generation function and into the class that actually knows what it needs.
* **Clean exporting** — `BaseVideo` received a new `export()` function that replaces the old `generate` logic.  It treats the incoming streams as opaque, simply renders them, and writes to the path provided by the object.
* **Path consistency** — moving forward, filename and folder logic will be encapsulated within the classes themselves, eliminating global folder variables and duplicated string formatting entirely.

The result: a pipeline where every artifact knows its own path, composes its own streams, and delegates only the final encode to a dumb, reliable render engine.
