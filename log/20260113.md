# 13 Jan 2026 – DeepSeek 3 – Declarative video composition with JSON timeline

_Studio_

A completely new video composition system was designed and implemented, replacing the patchwork of specialised scripts with a single, universal declarative engine.

### A declarative JSON format

The composition is described in a clean, minimal JSON schema.  Assets — videos, audios, images, fonts — are defined as arrays of file paths.  The timeline is a list of items, each with a type (`video`, `audio`, `image`, `layout`) and properties: `timestamp`, `duration`, `offset`, `loop`.  Effects like `fadein`, `fadeout`, `eq`, and `loudnorm` are attached per‑item and globally.  The format supports smart defaults: `null` generates black or silent media, and negative start times position elements from the end.

### Two core classes

* `Content` manages JSON I/O — loading, saving, and validating composition data.
* `Video` is the universal renderer: it takes JSON and produces MP4 or MP3 files through FFmpeg.

### Specialised input classes

`AudioInput` and `VideoInput` handle the complexity of source files.  They probe durations, generate silent or black filler, fit inputs to target dimensions, and — critically — count how many times a source is used and split the stream accordingly.  This solved the notorious stream‑reuse bug that had been plaguing complex compositions.

### What this enables

A single engine can now produce:
* Psalms videos with background visuals, text overlays, narration, and music.
* Torah portions with timed text displays and audio.
* Aramaic lessons with instructional slides and examples.
* Piyutim music videos with lyrics and artistic effects.
* Any combination of audio, video, image, and text with fades and normalisation.

The design principles were fanatical: minimal JSON, zero over‑engineering, strict separation of concerns (Content knows *what*, Video knows *how*), and declarative over imperative.  The test suite proves the system works across all content types.
