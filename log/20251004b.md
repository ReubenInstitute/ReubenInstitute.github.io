# 4 Oct 2025 – DeepSeek 3 – Coding standards: tabs, blank lines, naming

_General_

We formalised a coding philosophy grounded in a single, uncompromising principle: *a project is completed when there is nothing left to remove from it.*  That minimalist mantra drove every decision, producing a set of standards that are less a style guide and more a manifesto.

### The standards

**Indentation** — tabs only.  The argument is semantic, not aesthetic: a tab is a single unit of indentation, and using spaces to simulate tabs is visual formatting that belongs in the editor, not in the code.

**Blank lines** — only the bare minimum courtesy exceptions.  Blank lines inside functions are treated as visual crutches for poorly structured code; if a function feels like it needs them, refactor it instead.

**Variable naming** — context dictates length.  Single letters are fine in tight scopes where meaning is obvious; full words are required when the scope widens.  Abbreviations and redundant type suffixes (like `_number` appended to a variable that is already clearly a number) are rejected outright.  The philosophy borrows from mathematical notation: `c = book.chapter`, not `chapter_number`.

**String quotes** — double quotes for content that matters to the user (display strings, messages), single quotes for internal code.  A semantic distinction that lets the eye scan for user‑facing text instantly.

**LLM resistance** — explicit rejection of AI‑suggested anti‑patterns.  Language models are trained on codebases full of `get_` prefixes, `_num` suffixes, and other verbose conventions; those suggestions are ignored by default.

### The philosophical stances

* Against visual formatting: “Fix your editor, don’t mess up the code.”
* Against redundancy: if you need `_number` to know something is a number, your code is unclear.
* Pro semantic purity: every character must justify its existence.
* Minimalist extremism: we are not extending existing standards like PEP 8, but applying a fanatical philosophical overlay to any language’s conventions.

The result is a coherent, opinionated coding philosophy that prioritises semantic purity and minimalism — the rebellious minimalist cousin of traditional style guides.
