# 17 Dec 2025 – DeepSeek 3 – Text rendering and fake shadow fix

_Media_

The text on our TikTok videos was unreadable, and hunting down why exposed a tangle of architectural debt that went far deeper than a visual glitch.

### The real problems

Three things were wrong, and none of them were the shadow:
* Fake shadows — the code was duplicating text offset by 3 px and calling it a shadow.  That’s not a shadow; it’s a ghost.
* Wrong functions in wrong places — overlays were calling slide functions, slides were calling overlay functions, and nothing was where it belonged.
* Over‑engineering everywhere — variables for one‑time calculations, dead code branches that could never execute, and a general sense that the code had been written by someone trying to solve problems that didn’t exist.

### The rendering engine cleanup

We renamed `text` to `draw_text` with proper parameters: `(canvas, xy, text, font=None, color=None, shadow=True, anchor=None, direction=None)`.  The dead branch `if not extreme_chars:` — which was pointless and always false — was deleted.  Anchor handling was fixed so that `anchor="mm"` works correctly for overlays that need true centering.  Slide functions were renamed to `draw_*_slide()` to make their purpose explicit: they’re for verse content, nothing else.  Useless variables like `current_color`, `draw_y`, and `paste_x` were removed.

### Overlay and slide separation

The overlay classes — `EpisodeOverlay`, `ParashahOverlay`, `PsalmOverlay` — were stripped down to their essentials.  No more `Word` object usage; just strings.  No more bounding‑box calculations.  The fake logo shadow effect was killed.  Positions are hardcoded where they’re fixed: logo at (306, 100), text lines at fixed Y coordinates.  They use `draw_text()` with `anchor="mm"` for real centering.

The slide classes — `PsalmSlide`, `ParashahSlide` — were updated to use the `draw_*_slide()` functions and maintain verse‑highlighting logic (yellow for highlighted verses) with proper height calculation using `extreme_chars`.

### The shadow saga

We tried to replace the fake offset shadow with a real radial one.  Hours were spent fighting letter‑center filling — shadows pooling inside `ב`, `ה`, `ח` — and parameter hell: spread, blur, opacity, MaxFilter iterations.  The original shadow (4 px spread, 4× MaxFilter(3), GaussianBlur(3), 75% opacity) worked but had dark centers.  A simpler approach using just alpha‑channel blurring couldn’t find the balance between visibility and darkness.  The shadow problem remains unsolved, but the foundation around it is finally clean.

### The real lesson

We spent hours on a three‑line shadow problem that revealed thirty lines of architectural debt.  The fix wasn’t in the shadow algorithm — it was in cleaning up the code the shadow sat on top of.  Text is now properly positioned and readable.  The shadow needs a balanced, simple implementation that respects Hebrew letterforms.  But the architecture is no longer fighting itself.
