# 8 Oct 2025 – DeepSeek 3 – Bible system refactored to real Verse objects

_Code_

We had a quiet inconsistency eating away at the Bible content types.  Psalms used real `Bible.Verse` objects directly and everything worked.  Articles and Parashot wrapped verses in custom objects that looked similar but had no real connection to the Bible context, so properties like `verse.chapter.number` would silently fail.  The custom wrappers were losing the link to the canon.

The fix was to eliminate every custom `Verse` class and let all three content types use the same `Bible.Verse` objects directly.

### BibleArticles.py

The custom `Verse` class was removed entirely.  `Segment` now stores Bible verse objects instead of wrappers.  `Article.load()` works with `self.verses` directly, no intermediate list needed.  The `Segment` constructor parameter was renamed from `idx` to `number` to reflect what it actually holds.

### BibleParashot.py

The same pattern.  Custom `Verse` gone.  `Segment` and `ParashahEpisode.load()` both use Bible verses directly, and the intermediate variables that had been holding the wrappers were eliminated.

### BiblePsalms.py

This one was already correct — it was using `self.verses = chapter.verses` all along.  We simply removed the unnecessary complexity that had crept in around the edges.

### App updates

Both `app.py` and `studio.py` were updated to use Hebrew verse numbers consistently.  Templates now show `verse.hebrew_number` instead of Arabic numerals, and all existing functionality was preserved.

### The result

A consistent API across every Bible content type, with no more silent failures and no more wrappers:

```python
psalm_verse   = bible.psalms[0][0]
article_verse = bible.articles[0][0][0]
parashah_verse = bible.parashot[0][0][0][0]

# All three now work identically:
print(verse.chapter.number)    # Real chapter number
print(verse.hebrew_number)     # Hebrew formatting
print(verse.text)              # Formatted text
```

The architecture is circular‑import‑free, consistent everywhere, and maintainable because changes to `Bible.Verse` propagate to every content type automatically.  Less code, fewer classes, more reliability.
